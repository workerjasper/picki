# Picki — Claude Code 프로젝트 가이드

## 프로젝트 개요

**Picki**는 링크 + 한줄 코멘트를 공유하는 일상 트렌드 SNS 서비스입니다.
인스타그램이 사진을 공유하는 SNS라면, Picki는 링크를 공유하는 SNS입니다.

- URL을 붙여넣으면 OG 태그를 자동 파싱해 예쁜 카드로 생성
- 카테고리별 트렌드 랭킹 피드 제공
- 광고(애드센스 → 네이티브 광고 → 트렌드 리포트) 기반 수익 모델

---

## 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | Next.js (React) | Vercel 배포 최적화, SEO 대응 |
| 스타일 | Tailwind CSS | 빠른 UI 개발 |
| 백엔드/DB | Supabase | DB + 인증 + 어드민 대시보드 기본 제공 |
| 배포 | Vercel | GitHub push → 자동 무중단 배포 |
| OG 파싱 | Next.js API Route | 서버사이드에서 URL 메타 파싱 |

---

## 서비스 구조

```
사용자가 URL 입력
    ↓
서버에서 OG 태그 파싱 (제목, 설명, 썸네일)
    ↓
카드 자동 생성 + 카테고리 선택 + 한줄 코멘트
    ↓
피드에 노출 → 좋아요/저장/랭킹 반영
```

---

## 핵심 기능 (MVP)

### 반드시 있어야 하는 것
- [ ] URL 붙여넣기 → OG 파싱 → 카드 자동 생성
- [ ] 카테고리 선택 (재미, 맛집, 재테크, 건강, 게임, 꿀팁, 영상, 여행)
- [ ] 한줄 코멘트 입력
- [ ] 메인 피드 (오늘의 트렌드 랭킹)
- [ ] 좋아요 / 저장
- [ ] 회원가입 / 로그인 (이메일 + 카카오/구글 소셜)
- [ ] 내 프로필 (내가 올린 링크 목록)
- [ ] 저장한 링크 목록
- [ ] 비로그인 사용자도 피드 열람 가능

### 나중에 추가할 것
- 팔로우/팔로워
- 댓글
- 알림
- 검색
- 네이티브 광고 슬롯
- 주간 트렌드 리포트

## 백오피스 (Admin)

### 접근 경로
- URL: `/admin`
- 관리자 계정만 접근 가능 (Supabase `users` 테이블에 `is_admin boolean` 컬럼으로 관리)
- 비관리자 접근 시 `/` 로 리다이렉트

### 화면 목록

| # | 화면 | 경로 |
|---|------|------|
| 1 | 대시보드 (통계) | `/admin` |
| 2 | 회원 관리 | `/admin/users` |
| 3 | 링크 관리 | `/admin/links` |
| 4 | 신고 관리 | `/admin/reports` |

### 기능 상세

**대시보드 (`/admin`)**
```
- 오늘 가입자 수
- 오늘 업로드된 링크 수
- 총 회원 수
- 총 링크 수
- 카테고리별 링크 분포 (차트)
- 일별 가입자 추이 (최근 7일)
```

**회원 관리 (`/admin/users`)**
```
- 회원 목록 테이블 (닉네임, 이메일, 가입일, 링크 수, 상태)
- 검색 (닉네임 / 이메일)
- 회원 상세 보기 (올린 링크 목록, 활동 내역)
- 차단 / 차단 해제 처리
```

**링크 관리 (`/admin/links`)**
```
- 링크 전체 목록 테이블 (제목, 카테고리, 올린 사람, 좋아요 수, 등록일)
- 카테고리 필터
- 날짜 필터
- 링크 삭제
```

**신고 관리 (`/admin/reports`)**
```
- 신고된 콘텐츠 목록 (신고 대상, 신고 사유, 신고자, 신고일)
- 처리 상태 필터 (미처리 / 처리완료)
- 삭제 처리 (링크 삭제 + 신고 완료 처리)
- 무시 처리 (신고 완료 처리만)
```

### DB 추가 사항

**users 테이블에 컬럼 추가**
```
is_admin    boolean (default false)
is_banned   boolean (default false)
```

**reports 테이블 신규 생성**
```
id          uuid (PK)
link_id     uuid (FK → links)
reporter_id uuid (FK → users)
reason      text (신고 사유)
status      text (pending / resolved / ignored)
created_at  timestamp
```

## DB 테이블 구조 (Supabase)

### users
```
id          uuid (PK, auth.users 연동)
nickname    text
email       text
avatar_url  text
created_at  timestamp
```

### links
```
id          uuid (PK)
user_id     uuid (FK → users)
url         text
title       text
description text
thumbnail   text
site_name   text
category    text
comment     text (한줄 코멘트, max 60자)
like_count  int (default 0)
save_count  int (default 0)
created_at  timestamp
```

### likes
```
id          uuid (PK)
user_id     uuid (FK → users)
link_id     uuid (FK → links)
created_at  timestamp
UNIQUE(user_id, link_id)
```

### saves
```
id          uuid (PK)
user_id     uuid (FK → users)
link_id     uuid (FK → links)
created_at  timestamp
UNIQUE(user_id, link_id)
```

## 트렌드 랭킹 로직

- 기준: `like_count * 2 + save_count` (당일 생성된 링크 기준)
- 매일 자정 기준으로 리셋
- 카테고리별 필터링 가능

---

## 수익화 로드맵

```
1단계: 구글 애드센스 (MAU 확보 전)
    ↓
2단계: 네이티브 광고 카드 (MAU 1만 이후)
    ↓
3단계: 주간 트렌드 리포트 B2B 판매 (데이터 충분히 쌓인 후)
```

## 코딩 컨벤션

- 언어: TypeScript
- 컴포넌트: 함수형, default export
- 파일명: kebab-case (`link-card.tsx`)
- API: Next.js API Routes (`/pages/api/` 또는 `/app/api/`)
- 환경변수: `.env.local` + Vercel 환경변수로 관리
- Supabase client: `lib/supabase.ts` 에서 싱글턴으로 관리

---

## 시작하기

```bash
# 1. 프로젝트 생성
npx create-next-app@latest picki --typescript --tailwind --app

# 2. 의존성 설치
npm install @supabase/supabase-js @supabase/auth-helpers-nextjs

# 3. 환경변수 설정
# .env.local 에 아래 추가
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# 4. 개발 서버 실행
npm run dev
```
